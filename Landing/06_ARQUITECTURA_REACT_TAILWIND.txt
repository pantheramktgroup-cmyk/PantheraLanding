# ARQUITECTURA REACT + TAILWIND

## Objetivo técnico

Crear una landing mantenible, editable por contenido, visualmente premium y fácil de desplegar en Vercel.

## Stack

- React
- Vite
- Tailwind CSS
- GSAP
- @gsap/react
- ScrollTrigger
- Lenis
- SplitType

## Archivo único de contenido

Crear:

src/content/landingCopy.js

Este archivo debe contener todo el copy de la página en orden:

export const landingCopy = {
  nav: {},
  hero: {},
  video: {},
  transformation: {},
  audience: {},
  problem: {},
  apexSystem: {},
  systemPieces: {},
  comparison: {},
  testimonials: {},
  about: {},
  faq: {},
  booking: {},
  footer: {}
};

Regla:
Ningún texto de la landing debe estar hardcodeado en componentes.

## CTAs

Todos los CTAs deben apuntar a #booking.

Crear utilidad:

src/lib/scrollToBooking.js

Debe permitir scroll suave hasta la sección booking.

Si Lenis está activo, usar Lenis scrollTo.
Si no, usar element.scrollIntoView({ behavior: 'smooth' }).

## Componentes UI

### Button.jsx

Variantes:
- primary
- secondary
- ghost
- fullScreen

Debe aceptar:
- children
- href
- onClick
- className

Todos los botones de conversión deben disparar scroll a #booking.

### HorizontalScroll.jsx

Reusable para:
- ApexSystem
- Testimonials

Debe estar protegido para mobile/reduced motion.

### CTAFullScreen.jsx

Pantalla negra con frase grande y botón.
Usar al final de scrolls horizontales.

## Responsividad

Desktop:
- Experiencia cinematográfica.
- Scroll horizontal.
- Pinning.
- Layouts asimétricos.

Tablet:
- Mantener interacción, pero reducir distancias.

Mobile:
- Priorizar legibilidad.
- Evitar scroll horizontal complejo si afecta usabilidad.
- Cards apiladas verticalmente.
- CTAs visibles.
- Calendario usable.

## Accesibilidad

- Contraste alto.
- Botones con foco visible.
- No depender solo de animaciones para mostrar información.
- Usar headings semánticos.
- Iframe con title.
- Accordion accesible.
